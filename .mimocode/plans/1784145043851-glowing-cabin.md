# Plan: Share Question Sets by Academic Year + Module (Remove Semester Scope)

## Goal
Change question sets from being scoped per `(academic_year, semester, module)` to `(academic_year, module)`. One question set per academic year + module is shared across all semesters. Forms still track their own semester.

## Current State
- `feedback_question_sets` has columns: `academic_year_id`, `semester_id`, `module`
- Unique constraint: `(academic_year_id, semester_id, module)` — one set per year+semester+module
- Form creation AJAX filters by `(academic_year_id, semester_id, module)` to find a matching set
- Clone creates a copy for a specific target year+semester+module

## Target State
- `feedback_question_sets` has columns: `academic_year_id`, `module` (semester_id removed)
- Unique constraint: `(academic_year_id, module)` — one set per year+module
- Form creation AJAX filters by `(academic_year_id, module)` to find the shared set
- Clone copies all question sets from a source year to a target year (one per module)
- Forms still store their own `semester_id` in `feedback_forms` (unchanged)

---

## Step 1: Database Migration (`database/migrate_v15.sql`)

Create a new migration file:

1. **Handle existing data**: For each `(academic_year_id, module)` that has multiple question sets across semesters, keep the one with the most questions (or highest ID), and reassign/redirect any `feedback_forms` referencing the other sets to the kept one. Delete the redundant question sets and their orphaned questions.

2. **Drop the foreign key** on `feedback_question_sets.semester_id`:
   ```sql
   ALTER TABLE feedback_question_sets DROP FOREIGN KEY feedback_question_sets_ibfk_2;
   ```

3. **Drop the `semester_id` column**:
   ```sql
   ALTER TABLE feedback_question_sets DROP COLUMN semester_id;
   ```

4. **Update the unique constraint**:
   ```sql
   ALTER TABLE feedback_question_sets DROP INDEX uq_question_set;
   ALTER TABLE feedback_question_sets ADD UNIQUE KEY uq_question_set (academic_year_id, module);
   ```

5. **Update indexes**: Drop `idx_fqs_ay_sem`, add `idx_fqs_ay_mod (academic_year_id, module)`.

6. **Update `schema.sql`** to reflect the new table structure.
