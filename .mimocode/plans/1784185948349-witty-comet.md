# Fix: Question Set Creation — default_question_templates Table Missing

## Root Cause Analysis

The error `Table 'studentfeedbackintern.default_question_templates' doesn't exist` occurs because:

1. **Migration file has wrong database name**: `database/migrate_v16.sql` line 6 says `USE studentfeedback;` but the actual database is `studentfeedbackintern` (confirmed in `config/db.php` line 5).

2. **Migration was never executed**: The SQL file exists but was never run against the MySQL server.

3. **PHP code queries the missing table**: `admin/question_sets.php` line 104 runs `SELECT COUNT(*) FROM default_question_templates` inside a transaction. When the table doesn't exist, MySQL throws an error. The error is caught by the `catch` block at line 152 which rolls back the transaction and shows the error message.

### 2. Fix `admin/question_sets.php`

Replace the current template-query block (lines 103-132) with resilient logic:

```php
// Check if default_question_templates table exists
$tblCheck = $conn->query("SHOW TABLES LIKE 'default_question_templates'");
$hasTable = ($tblCheck && $tblCheck->num_rows > 0);

if ($hasTable) {
    // Try to copy from default templates
    $tpl = $conn->prepare("SELECT question_no, question_text, question_type, options_json FROM default_question_templates WHERE module=? ORDER BY question_type, question_no ASC");
    $tpl->bind_param('s', $module);
    $tpl->execute();
    $srcQuestions = $tpl->get_result()->fetch_all(MYSQLI_ASSOC);
    $tpl->close();
} else {
    $srcQuestions = [];
}

// Fallback: if no templates found or table doesn't exist, copy from most recent set
if (empty($srcQuestions)) {
    $tpl = $conn->prepare("SELECT id FROM feedback_question_sets WHERE module=? AND status='active' AND id != ? ORDER BY id DESC LIMIT 1");
    $tpl->bind_param('si', $module, $newSetId);
    $tpl->execute();
    $tplRow = $tpl->get_result()->fetch_assoc();
    $tpl->close();

    if ($tplRow) {
        $tplId = (int)$tplRow['id'];
        $srcQ = $conn->prepare("SELECT question_no, question_text, question_type, options_json FROM feedback_questions WHERE question_set_id=? ORDER BY question_no ASC");
        $srcQ->bind_param('i', $tplId);
        $srcQ->execute();
        $srcQuestions = $srcQ->get_result()->fetch_all(MYSQLI_ASSOC);
        $srcQ->close();
    }
}
```

Key change: `SHOW TABLES LIKE 'default_question_templates'` runs first. If the table doesn't exist, it returns 0 rows and we skip the template query entirely. No error is thrown. The fallback (copy from most recent set) handles this gracefully.
